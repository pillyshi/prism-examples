# JMED-LLM SMDIS 解析メモ

実行日: 2026-04-18  
データセット: JMED-LLM / SMDIS（1,920件 × 8タグ）  
Prism: 0.3.1 / NLIモデル: MoritzLaurer/mDeBERTa-v3-base-mnli-xnli

---

## データセット構造

SMDIS は日本語SNS投稿に対して8種類の症状タグを付与したデータセット。
1,920件の投稿が各タグについて独立にラベル付けされており、各タグの正例率は約9〜15%。

| タグ | 正例数 | 負例数 | 正例率 |
|---|---|---|---|
| diarrhea | 182 | 1738 | 9.5% |

**構造的な問題点**  
同一投稿が全8タグに対してラベルを持つため、あるタグの「負例」は他タグの「正例」テキストを多く含む。
たとえばdiarrheaの負例には、発熱・咳・鼻水に関する投稿が大量に混在する。

---

## 事前定義軸による分析（analyze_smdis.py）

軸のhypothesisをNLIスコアリングに用いたところ、**正例よりも負例でNLIスコアが高くなる逆転現象**が発生した。

原因として以下が挙げられる：

1. **時制・人称の問題**  
   SMDIS のラベル基準は「投稿日から1日以内」の症状有無。「ネパールで下痢になった（過去）」はdiarrhea=noとラベルされるが、NLIは時制を区別できず「下痢への言及」を検出してしまう。

2. **タグ間クロス汚染**  
   diarrheaの負例に含まれる風邪・インフルエンザ関連テキストが、「体調不良」「症状の言及」系の特徴量を汚染する。NLIは「この投稿は症状を示している」に対し、負例でも高スコアを返す。

3. **仮説の粒度**  
   「この投稿は下痢であることを示している」という仮説は正例に厳しく、負例にも当てはまりやすい。

## 軸の自動発見（discover_smdis.py）

ラベルを一切使わずにテキストのみからPrismに軸を自動発見させた結果、10軸が生成された。

```
発見された軸:
  風邪の症状の強さ / インフルエンザの可能性 / 体調不良の認識
  下痢の症状 / 花粉症の症状 / 感情の表出
  休暇の理由 / 薬の使用 / 自宅療養の必要性 / 社会的エチケット
```

**注目点: ラベルなしで「下痢の症状」軸が自然に出現した。**  
このデータセットにおいてdiarrheaが統計的に識別可能な変動軸であることをPrismが自律的に発見した。

### 各軸のGT一致率とCV スコア

| 軸 | acc | f1 (GT) | cv_score |
|---|---|---|---|
| 下痢の症状 | 0.548 | 0.256 | 0.865 |
| 休暇の理由 | 0.573 | 0.177 | 0.822 |
| 薬の使用 | 0.845 | 0.178 | 0.992 |
| インフルエンザの可能性 | 0.760 | 0.199 | 0.758 |
| 風邪の症状の強さ | 0.736 | 0.199 | 0.727 |
| 花粉症の症状 | 0.736 | 0.369 | 0.669 |
| 自宅療養の必要性 | 0.801 | 0.186 | 0.695 |
| 感情の表出 | 0.866 | 0.176 | 0.502 |
| 体調不良の認識 | 0.872 | 0.179 | 0.415 |
| 社会的エチケット | 0.905 | 0.173 | 0.000 |

「下痢の症状」軸のcv_score=0.865は高いが、GTとのf1=0.256は低い。  
NLIラベルとGTが逆相関している（acc=0.548 ≈ 逆向きの二値分類）ことを示す。  
これは事前定義軸と同様の時制・人称問題が自動発見軸でも再現されたことを意味する。

### 注目すべき特徴量

**「下痢の症状」軸の選択特徴量:**

```
[+7.967] 下痢の英語表記を含むか   ← 最大係数
[+4.426] 下痢の症状が明示されているか
[+3.056] お腹の痛みを言及しているか
```

「下痢の英語表記（diarrhea）を含むか」が係数7.967と突出している。  
英単語"diarrhea"を含む投稿が正例に強く偏っていることを示す。  
医療用語を英語で表記するSNS投稿（例: 「diarrheaになった」）は現在時制で自身の症状を述べている可能性が高く、時制問題を回避できる特徴量といえる。

**「薬の使用」軸:**

```
[+18.505] 薬の使用を示唆する
```

1特徴量・係数18.5でcv=0.992は過学習。「薬の使用」は症状の種類を問わず一般的な特徴であり、diarrheaを識別しているのではなく薬言及の有無を学習している。

## まとめと考察

Prismのaxis discoveryはラベルなしでデータセットの主要な変動軸を適切に抽出できる。  
しかしSMDISのような「症状の有無」タスクでは、NLIベースのラベリングが時制・人称を考慮できないため、  
自動発見された軸のNLIラベルがGTと一致しにくい。

Prismが本来力を発揮するのは、テキストの**表現スタイル・トーン・語彙の傾向**を軸として発見するユースケースであり、  
「特定の医療事実が現在進行形で成立しているか」という細粒度の事実判断とは相性が悪い。

SMDISへの適用として現実的な方向性：

1. **時制・人称を考慮した仮説設計**  
   例: 「このテキストは投稿者自身が現在下痢であることを述べている」
2. **discoverした軸の解釈的活用**  
   GT予測ではなく、「どのような言語的パターンがデータセットに存在するか」の探索として使う
3. **他タグへの適用**  
   花粉症（hayfever）は他タグと語彙的重複が少なく、より良い結果が期待できる可能性がある
